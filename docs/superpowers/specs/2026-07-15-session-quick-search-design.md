# 会话快速搜索设计

## 目标

提供一个全局会话快速搜索入口。用户按下 `Ctrl+F` 后，可以快速检索会话，并通过双击或按下 `Enter` 建立 SSH 连接。

## 范围

- 新增全局会话搜索弹框。
- 支持按会话名称、主机、用户名和分组名称检索。
- 支持鼠标双击和键盘导航后按 `Enter` 连接。
- 复用现有会话连接流程，不修改后端 API、数据模型或持久化格式。
- 不扩展为通用命令面板，不加入会话编辑、删除或分组管理操作。

## 快捷键规则

- Windows 和 Linux 使用 `Ctrl+F`，macOS 同时支持 `Command+F`。
- 终端获得焦点时，快捷键仍打开会话快速搜索，并阻止按键继续发送给终端。
- 焦点位于普通 `input`、`textarea`、`select` 或可编辑元素时，不拦截快捷键，保留表单内原生查找或输入行为。
- 弹框已经打开时再次按下快捷键，保持弹框打开并聚焦搜索输入框。
- `Escape` 关闭弹框。

## 交互设计

弹框使用 shadcn/ui 的 `Dialog` 风格，宽度适合快速浏览，不占满应用窗口。顶部展示标题和快捷键提示，中部为自动聚焦的搜索输入框，底部为可滚动结果列表。

结果项展示：

- 会话名称作为主信息。
- `username@host:port` 作为连接信息。
- 所属分组名称作为辅助信息；没有分组时显示“未分组”。

打开弹框后默认展示所有会话并选中第一项。输入查询内容后立即在前端过滤，匹配不区分大小写。键盘 `ArrowUp` 和 `ArrowDown` 在结果间循环移动选择，`Enter` 连接当前选中项；鼠标移动或单击可更新选中项，双击立即连接。

当过滤结果变化时，选择自动回到第一项，避免保留已经不存在的索引。没有会话时显示“暂无会话”，有会话但无匹配结果时显示“未找到匹配会话”。

## 组件边界

### `SessionQuickSearchDialog`

独立负责弹框展示、过滤、选中状态、键盘导航和触发连接。组件接收 `sessions`、`folders`、`open`、`onOpenChange` 和 `onConnect`，不直接读取后端服务或应用 Store。

### `App`

继续作为全局快捷键入口。快捷键处理只负责派发打开搜索弹框的应用事件，避免将 Context 数据耦合进模块级键盘处理函数。

### `SessionWorkspaceProvider` 内部挂载点

新增轻量宿主组件，通过 `useSessionWorkspace()` 获取会话、分组和现有 `connect` 方法，并渲染 `SessionQuickSearchDialog`。连接动作完全复用 `workspace.connect(session.id)`。

## 数据流

1. 用户按下 `Ctrl+F` 或 `Command+F`。
2. 全局快捷键处理阻止浏览器和终端默认行为，并派发打开事件。
3. 搜索弹框宿主监听事件并打开弹框。
4. 用户输入查询，组件在内存中计算匹配结果。
5. 用户双击结果或按下 `Enter`。
6. 弹框先关闭，再调用现有 `workspace.connect(session.id)`。
7. 原有连接对话框继续处理连接中、主机指纹确认、失败和重试状态。

## 错误与并发处理

- 搜索组件不复制连接错误处理，连接失败继续由现有 `ConnectDialog` 展示。
- 如果已有连接流程正在处理，现有 `workspace.connect` 会给出提示，搜索弹框仍关闭，避免两个模态弹框叠加。
- 会话列表加载失败继续由工作区现有错误处理负责；搜索弹框只基于当前可用数据展示结果。

## 可访问性

- 弹框具有明确标题和说明。
- 结果区域使用列表框语义，结果项使用选项语义并暴露选中状态。
- 键盘可完成打开、输入、选择、连接和关闭的完整流程。
- 当前选中项使用主题令牌突出显示，不依赖固定颜色。

## 测试策略

- 快捷键测试：终端焦点下打开并阻止默认行为；普通表单输入中不拦截；无修饰键时不触发。
- 过滤测试：名称、主机、用户名、分组名称和大小写不敏感匹配。
- 交互测试：默认选择、上下方向键循环、结果变化重置选择、`Enter` 连接、双击连接和 `Escape` 关闭。
- 状态测试：空会话列表、无匹配结果和分组缺失展示。
- 集成测试：连接动作使用现有工作区 `connect`，不会新增后端调用路径。

## 验收标准

- 在非表单区域或终端中按 `Ctrl+F` 可稳定打开会话快速搜索。
- 搜索输入自动获得焦点，结果能按四类字段正确过滤。
- 用户可用鼠标双击或键盘选择后按 `Enter` 建立连接。
- 快捷键不会在普通输入框和设置表单中造成干扰。
- 新增行为有自动化测试看护，前端测试、覆盖率、lint、格式化和 `wails3 build` 全部通过。
