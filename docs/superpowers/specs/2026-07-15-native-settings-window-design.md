# Wails3 原生设置窗口设计

## 目标

将当前嵌入主窗口的 React 设置弹框迁移为 Wails3 独立原生窗口，消除前端模拟拖动带来的窗口边界限制、拖动残影和平台不稳定行为，同时保留设置修改时对主窗口的实时预览能力。

## 已确认的交互边界

- 设置窗口使用无边框 Wails 原生窗口和 React 自定义标题栏。
- 自定义标题栏视觉与当前设置弹框标题栏保持一致。
- 设置窗口允许缩放。
- 设置窗口为非模态窗口，打开后仍可操作主窗口。
- 全局最多存在一个设置窗口。
- 标题栏拖动必须使用 Wails 原生拖动区域，不允许使用 JavaScript 坐标变换模拟窗口移动。

## 方案选择

采用懒加载原生单例窗口：第一次点击设置时创建窗口，窗口存在时再次点击只显示并聚焦，关闭后销毁，下次打开重新创建并读取最新数据。

不在应用启动时预创建隐藏窗口，避免增加启动耗时、常驻内存和隐藏窗口中的陈旧状态。不允许设置窗口加载完整主应用，避免重复初始化会话、终端、文件传输和全局事件桥。

## Go 窗口控制器

新增独立的设置窗口控制器，由它统一处理窗口创建、查找、聚焦、定位和关闭。

控制器职责：

- 使用固定名称 `settings` 查询窗口。
- 使用互斥锁串行化“查询后创建”，防止快速连点生成多个窗口。
- 已存在窗口执行 `Show()` 和 `Focus()`。
- 不存在时使用固定窗口参数创建。
- 主窗口关闭时关闭仍存在的设置窗口。
- 设置窗口关闭时通知主窗口取消未保存预览。

控制器通过小型窗口注册表接口隔离 Wails 具体类型，单元测试使用假窗口验证单例和生命周期，不启动 GUI。

## 窗口参数

- 名称：`settings`
- 标题：`MSSH 设置`
- URL：`/?window=settings`
- 初始尺寸：`980 × 720`
- 最小尺寸：`820 × 600`
- 原生标题栏：关闭，即 `Frameless: true`
- 可缩放：启用
- 始终置顶：关闭
- 文件拖放：关闭
- 背景类型：实体背景，避免透明合成导致设置窗口残影
- 初始位置：主窗口所在显示器的工作区内，优先居中覆盖主窗口
- Windows：启用 `NonClientRegionSupport`，不启用 `WebView2CompositionHosting`
- Linux：保持 `WebviewGpuPolicyNever`，降低 WebView 合成不稳定风险

窗口位置根据主窗口位置和尺寸计算，并限制在当前显示器 `WorkArea` 范围内。无法取得主窗口显示器信息时退化为 Wails 的屏幕居中能力。

## 自定义标题栏

新增 `SettingsWindowTitleBar`，使用当前设置弹框标题区域的视觉语言：

- 高度、边框、背景、标题字号和间距与现有设置弹框一致。
- 左侧显示“设置”。
- 右侧只保留关闭按钮，不增加最小化和最大化按钮。
- 标题文字和空白区域使用 `[--wails-draggable:drag]`。
- 关闭按钮使用 `[--wails-draggable:no-drag]` 并调用当前窗口的 `Window.Close()`。
- 不再使用 `useDraggableDialog`、Pointer Move、CSS Transform 或 React 状态计算窗口坐标。

标题栏只负责绘制和声明原生拖动命中区域；实际窗口移动由 Wails `startDrag` 和操作系统窗口管理器完成。因此标题栏仍可完全匹配应用设计，同时不会受到主窗口 DOM 边界限制。

设置窗口边缘缩放继续由 Wails 无边框窗口实现。窗口阴影、外部圆角和边缘命中范围可能因操作系统或桌面环境而略有差异，但客户区内的标题栏视觉保持一致。

## 打开命令

主窗口设置按钮不再派发浏览器 `CustomEvent`，改为通过 Wails Events 发出无参数命令：

```text
window:open-settings
```

Go 应用注册该事件并调用设置窗口控制器。窗口创建不暴露新的数据库或业务 API，也不允许前端传入任意 URL、尺寸或窗口参数。

## 独立前端入口

`frontend/src/main.tsx` 根据 `window.location.search` 选择根组件：

- 默认入口：启动主应用事件桥并渲染 `App`。
- `window=settings`：只渲染 `SettingsWindowApp`。

设置窗口不会创建 `SessionWorkspaceProvider`，不会初始化会话列表、终端池、SFTP、录制、宏和主窗口标题栏。

两个入口继续共享全局 CSS、shadcn/ui 组件、Tooltip Provider 和 Wails bindings。

## 设置页面结构

将 `SettingsDialog` 拆分为不依赖 Dialog 的 `SettingsView`。它保留当前纵向分类导航及以下分类：

- 通用
- 终端
- 密钥
- 同步
- 关于

`SettingsWindowApp` 负责加载设置 Hook、组装 `SettingsView` 参数、展示 Toast，并提供包含 `SettingsWindowTitleBar` 的全窗口布局。

移除以下主窗口职责：

- `SettingsDialog` 挂载
- `useDraggableDialog` 设置页拖动逻辑
- `settingsOpen` React 状态
- 主窗口 Sidebar 对字体列表、密钥和同步配置的提前加载

## 通用设置运行时拆分

当前 `useSettings` 同时承担通用设置、字体列表、密钥和同步配置。迁移后将通用设置加载与运行时应用拆为独立 Hook：

- 主窗口只加载字体、透明度、终端行为和终端池等运行时必需项。
- 设置窗口额外加载系统字体、密钥和同步配置。
- 设置窗口保存通用设置后，仍由现有后端服务持久化和调整终端池大小。

该拆分减少主窗口启动时不必要的字体枚举、密钥查询和同步配置查询。

## 跨窗口事件协议

### 通用设置预览

设置窗口修改界面字体或透明度时：

1. 在设置窗口本地应用预览。
2. 广播 `settings:general-preview`，携带字体、字号和透明度预览值。
3. 主窗口接收后立即应用相同 CSS 令牌。

### 通用设置保存

保存成功后广播：

```text
settings:general-changed
```

载荷为规范化后的完整通用设置。主窗口立即更新：

- 界面主字体与 Fallback 字体
- 界面字号
- 主窗口透明度
- 鼠标右键行为
- 选择即复制
- 最大终端池大小

### 取消未保存预览

设置窗口发生原生关闭时，Go 控制器广播：

```text
settings:preview-cancelled
```

主窗口重新读取数据库中的已保存通用设置，确保关闭按钮、系统快捷键关闭和窗口管理器关闭都不会残留预览值。

### 终端主题同步

设置窗口完成主题配置、主题导入、主题编辑、删除或内置主题重置后，广播：

```text
theme:catalog-changed
```

载荷包含设置窗口已经加载完成的主题目录快照。主窗口直接更新主题 Store 并重新计算现有终端的有效主题，避免再次读取数据库。

### 界面模式同步

任意窗口成功切换 Dark/Light Mode 后广播：

```text
appearance:color-mode-changed
```

两个窗口均更新根节点主题类和本地主题 Store。事件只在数据库保存成功后广播，失败时维持原有回滚逻辑。

## 生命周期与错误处理

- 设置窗口创建命令无用户输入参数，不允许任意窗口注入。
- 自定义标题栏的可交互控件必须排除在 Wails 拖动区域之外。
- 快速重复点击由 Go 互斥锁保证单例。
- 窗口不存在或已经销毁时重新创建，不保留失效窗口指针。
- 设置保存错误继续由设置窗口 Toast 展示，不向主窗口广播成功事件。
- 预览事件失败不阻塞用户编辑；保存仍以数据库调用结果为准。
- 主窗口关闭时主动关闭设置窗口，避免只剩设置窗口导致应用进程继续运行。
- 设置窗口关闭后由 Wails 从窗口注册表移除，控制器不维护长期窗口指针。

## 测试策略

### Go 测试

- 第一次打开创建指定参数的窗口。
- 重复打开只显示和聚焦现有窗口。
- 并发打开只创建一个窗口。
- 设置窗口位置正确居中并限制在工作区内。
- 主窗口关闭会关闭设置窗口。
- 设置窗口关闭广播取消预览事件。

### 前端测试

- `main.tsx` 根据 URL 参数选择正确入口。
- 主窗口设置按钮发送 Wails 打开命令。
- 主窗口不再渲染设置 Dialog 或加载设置专用数据。
- `SettingsView` 保留现有分类与全部保存行为。
- `SettingsWindowTitleBar` 使用 Wails 拖动区域并通过运行时关闭当前窗口。
- 设置窗口预览字体和透明度时广播预览事件。
- 保存通用设置后广播规范化配置。
- 关闭取消事件使主窗口恢复持久化配置。
- 主题目录和 Dark/Light Mode 在两个窗口间同步。
- 设置窗口入口不会初始化会话与终端工作区。

## 验收标准

- 点击设置按钮打开操作系统管理的独立设置窗口。
- 设置窗口标题栏与当前设置弹框标题栏视觉一致。
- 设置窗口可自由拖出主窗口范围，不出现前端拖动残影。
- 重复点击设置按钮不会创建多个窗口，并会将现有窗口带到前台。
- 设置窗口关闭后可再次正常打开并读取最新数据。
- 字体、透明度和主题修改能实时反映到主窗口。
- 未保存预览在任何原生关闭路径下都能恢复。
- 主窗口启动时不再提前加载设置窗口专用数据。
- 前端测试覆盖率和 Go 总体覆盖率均不低于 90%。
- `goimports-reviser`、`golangci-lint`、前端构建和 `wails3 build` 全部通过。
