# 终端主题编辑器商业化重布局设计

## 目标

将设置左侧“外观”分类重命名为“终端”，明确该页面只配置终端主题。采用“实时终端预览 + 专业属性检查器”双栏布局，并将可扩展的预设主题收纳到可搜索下拉框。

## 页面结构

- 设置导航项由“外观”改为“终端”，内部 Tab 值同步改为 `terminal`。
- 页面顶部显示“终端主题”标题、用途说明和可搜索主题预设 Combobox。
- 主体采用双栏：左侧预览与 ANSI 调色板，右侧基础颜色、字体与光标属性。
- 底部提供明确的“保存主题”主操作。

## 主题预设下拉框

- 使用 shadcn `Combobox`，支持按主题名称搜索。
- 触发器显示当前主题名称和代表性色板摘要。
- 下拉项展示主题名称、背景色和 4–6 个 ANSI 代表色。
- 选择预设立即更新编辑草稿和页面内终端预览，但只有点击保存才写入设置并影响实际终端。
- 预设数据与编辑器组件分离，后续增加大量主题时不扩大主组件复杂度。

## 终端预览

- 使用真实终端语义内容：登录提示、命令、目录、成功、警告、错误和光标。
- 预览应用背景、前景、光标、字体、字号及 ANSI 颜色。
- 左侧预览在编辑区域滚动时保持可见。
- 预览下方显示当前字体和光标摘要。

## 属性检查器

- “基础颜色”Card：背景色、前景色、光标色。
- “字体与光标”Card：字体栈、字号、光标样式。
- 使用 shadcn `FieldGroup`、`Field`、`FieldLabel` 和现有输入控件。
- 颜色字段由颜色选择器与十六进制输入组合，保留精确编辑能力。

## ANSI 调色板

- 展示 16 个可点击色块，分为标准色和高亮色。
- 当前选中色块有明确焦点和选中状态。
- 只显示一个当前颜色编辑器，避免同时呈现 16 个十六进制输入造成视觉噪声。
- 每个颜色保留名称与索引，修改仍更新原 `ansi` 数组。

## 状态与保存

- `theme` 属性变化时同步重置编辑草稿，避免设置重新加载后仍显示旧值。
- 预设选择、颜色、字体、字号和光标修改只更新本地草稿。
- 点击保存调用现有 `onSave(TerminalTheme)`，不改变后端设置模型和终端主题数据结构。

## 组件拆分

- `terminalThemePresets.ts`：预设数据和匹配辅助函数。
- `ThemePresetCombobox.tsx`：可搜索预设选择。
- `TerminalThemePreview.tsx`：终端预览。
- `AnsiPaletteEditor.tsx`：ANSI 16 色编辑。
- `ThemeEditor.tsx`：表单状态和布局编排。

## 测试

- 设置导航显示“终端”且不再显示“外观”。
- Combobox 搜索和预设选择更新草稿。
- 终端预览应用主题颜色、字体和光标。
- ANSI 单色编辑只修改目标索引。
- 保存提交完整主题。
- 外部主题变化同步表单。
- 前端全测、构建、Go race/覆盖率、lint 和 `wails3 build`。
