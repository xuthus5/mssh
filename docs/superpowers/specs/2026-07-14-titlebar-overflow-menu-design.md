# 标题栏标签溢出菜单设计

## 目标

1. “打开标签列表”按钮移到标题栏右侧，位于 dark/light mode 按钮左侧。
2. 只有动态 SSH/回放标签实际超出可视区域、需要横向滚动时才显示该按钮。
3. 标签未溢出时，中间未被标签占用的区域可用于拖动应用窗口。
4. 窗口缩放、导航栏收缩、标签增删和字体变化后，按钮状态自动重新计算。

## 布局

标题栏从左到右排列：

1. 导航收缩按钮与固定“会话/宏”标签。
2. 按内容宽度展开、空间不足时收缩并横向滚动的动态终端标签条。
3. 至少保留一段稳定宽度的窗口拖拽区域。
4. 条件显示的“打开标签列表”按钮。
5. dark/light mode、设置和窗口控制按钮。

动态标签条不能继续使用 `flex-1` 抢占所有中间空间。它以内容宽度为基础参与收缩，拖拽区域使用 `flex-1` 接收剩余空间并设置最小宽度。

## 溢出检测

动态标签条通过真实 DOM 尺寸判断：

```ts
scrollWidth > clientWidth
```

使用 `ResizeObserver` 监听标签容器尺寸变化，并在标签集合变化后重新计算。组件通过 `onOverflowChange(boolean)` 将结果通知标题栏；无标签或组件卸载时恢复为 `false`。

不使用标签数量阈值，因为窗口宽度、标题长度、字体和缩放比例都会改变实际容量。

## 组件边界

- `DynamicTabStrip`：只渲染可滚动标签条、检测溢出、通知标题栏。
- `DynamicTabOverflowMenu`：独立渲染现有标签列表下拉菜单，复用 Store 中的标签和激活逻辑。
- `WindowTitleBar`：维护 `tabsOverflow` 状态，决定菜单是否放在右侧操作区显示。

## 测试

- 标签未溢出时不显示“打开标签列表”。
- `scrollWidth > clientWidth` 后显示按钮，恢复可容纳后隐藏。
- 按钮位于窗口拖拽区域之后、主题切换按钮之前。
- 从溢出菜单选择标签仍能激活并聚焦目标终端。
- 动态标签条不再使用 `flex-1`，拖拽区域具有稳定最小宽度。

## 验收标准

1. 建立少量 SSH 连接时不会立即出现标签列表按钮。
2. 只有标签条出现横向滚动需求时按钮才出现。
3. 按钮固定在 dark/light mode 左侧，不再位于标签条中间。
4. 标签左侧之后的剩余空白能够拖动窗口。
5. 窗口缩放和标签增删后按钮状态及时更新。
