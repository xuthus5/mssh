# 会话 CSV 导入导出设计

## 目标

会话总览提供独立的 CSV 导入导出能力，只处理 SSH 会话及其资产归属，不复用设置页的 `.msshbackup` 应用配置备份。

## 交互

- 页面头部提供“导入”和“导出”按钮，与“创建”并列。
- 导出支持全部会话或当前已选会话；默认不包含密码。
- 用户主动选择包含密码时，界面明确提示 CSV 将以明文保存。
- 导入支持“跳过重复项”和“覆盖重复项”，默认跳过。
- 导入结束展示逐行结果；成功后刷新分组、会话和资产分类。

## CSV 契约

- 编码：UTF-8 with BOM，兼容常见表格软件。
- 格式版本：`format_version=1`。
- 表头：`format_version,name,host,port,username,auth_method,password,key_name,key_public_key,folder_path,environment,project,tags,notes,keep_alive,term_type`。
- `folder_path` 与 `tags` 使用 JSON 字符串数组，避免名称包含逗号、斜杠或分号时产生歧义。
- 密钥只导出名称与公钥，不导出私钥；导入时优先按公钥匹配，其次按唯一名称匹配。
- 重复会话按 `name + host + port + username` 识别。
- 缺失的分组、环境、项目和标签在单行事务内自动创建。

## 安全边界

- 默认不导出密码；密码仅在用户主动确认后写入 CSV。
- 导出文件权限为 `0600`，采用同目录临时文件原子替换。
- 导入文件限制为 10 MiB、10000 条数据行。
- 使用 `encoding/csv` 与参数化 SQL，不拼接用户输入。
- 认证方式为密钥但本地无法匹配密钥时，该行失败，不创建不可用会话。

## EARS 验收项

- 当用户导出会话时，系统应生成版本化 UTF-8 CSV，并保持会话、分组与资产元数据完整。
- 当用户未启用“包含密码”时，系统应将所有密码单元格留空。
- 当用户仅选择部分会话导出时，系统应只写入所选会话。
- 当用户导入缺失分类的会话时，系统应自动创建对应分组、环境、项目和标签。
- 当用户选择跳过重复项时，系统应保留本地会话并报告跳过。
- 当用户选择覆盖重复项时，系统应更新本地会话；CSV 密码为空时应保留已有密码。
- 当 CSV 行无效或密钥无法匹配时，系统应仅标记该行失败，并继续处理其他有效行。
- 当导入文件超过安全限制或 CSV 结构无效时，系统应拒绝整个文件且不写入数据。
