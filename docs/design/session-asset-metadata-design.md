# 会话资产元数据设计

状态：已实现并验收
日期：2026-07-17

## 1. 背景

当前会话包含 `environment`、`project`、`tags`、`notes` 四个资产字段，但除创建、编辑、数据库持久化和同步备份外，没有进入资产展示、搜索、筛选、详情或批量管理流程。

本设计将这些字段从自由文本升级为可治理、可搜索、可筛选、可批量操作的正式资产维度。

项目仍处于 POC 阶段，不保留旧数据格式，不增加迁移与兼容代码，数据库直接采用最终结构。

## 2. 已确认决策

1. 分组保留为层级目录，一个会话属于一个分组。
2. 环境升级为全局单选字典。
3. 项目升级为全局单选字典。
4. 标签升级为全局多选字典，通过关联表绑定会话。
5. 备注保留为会话自由文本。
6. 环境、项目、标签和备注全部可选。
7. 环境、项目和标签允许在会话表单中快捷创建，并在资产中心统一治理。
8. 环境和标签使用受控语义色板，不允许任意 HEX 颜色。
9. 同类筛选条件使用 OR，不同类别之间使用 AND。
10. 普通搜索不包含备注；备注仅在高级筛选中显式启用。
11. 环境和项目的删除必须迁移或清空关联；标签删除必须展示影响并确认。
12. 所有重命名、迁移和删除操作必须由后端事务完成并写入审计日志。

## 3. 领域语义

| 维度 | 语义 | 示例 |
|---|---|---|
| 分组 | 层级位置或管理目录 | 华东机房 / 数据库 |
| 环境 | 生命周期与风险等级 | 生产、预发布、测试、开发 |
| 项目 | 业务或系统归属 | 支付平台、监控平台 |
| 标签 | 横向能力或任意特征 | MySQL、核心、Linux、值班 |
| 备注 | 无法结构化的补充信息 | 负责人、变更窗口、注意事项 |

分组不得替代环境或项目。一个会话只能属于一个分组、一个环境和一个项目，但可以拥有多个标签。

## 4. 最终数据模型

### 4.1 环境表 `asset_environments`

| 字段 | 类型 | 约束 |
|---|---|---|
| `id` | INTEGER | 主键，自增 |
| `name` | TEXT | 非空，最长 64 字符 |
| `name_key` | TEXT | 去除首尾空格并转小写，全局唯一 |
| `color_token` | TEXT | 非空，必须属于受控色板 |
| `sort_order` | INTEGER | 非空，默认 0 |
| `created_at` | TEXT | 非空 |
| `updated_at` | TEXT | 非空 |

### 4.2 项目表 `asset_projects`

| 字段 | 类型 | 约束 |
|---|---|---|
| `id` | INTEGER | 主键，自增 |
| `name` | TEXT | 非空，最长 64 字符 |
| `name_key` | TEXT | 规范化后全局唯一 |
| `code` | TEXT | 可空，最长 24 字符 |
| `code_key` | TEXT | 可空，非空时代号忽略大小写唯一 |
| `description` | TEXT | 非空，默认空字符串，最长 500 字符 |
| `sort_order` | INTEGER | 非空，默认 0 |
| `created_at` | TEXT | 非空 |
| `updated_at` | TEXT | 非空 |

项目代号用于列表和快速搜索结果中的紧凑展示。项目不配置颜色，避免与环境和标签产生视觉竞争。

### 4.3 标签表 `asset_tags`

| 字段 | 类型 | 约束 |
|---|---|---|
| `id` | INTEGER | 主键，自增 |
| `name` | TEXT | 非空，最长 32 字符 |
| `name_key` | TEXT | 规范化后全局唯一 |
| `color_token` | TEXT | 非空，必须属于受控色板 |
| `created_at` | TEXT | 非空 |
| `updated_at` | TEXT | 非空 |

### 4.4 会话标签关联表 `session_tags`

| 字段 | 类型 | 约束 |
|---|---|---|
| `session_id` | INTEGER | 外键，引用 `sessions.id`，级联删除 |
| `tag_id` | INTEGER | 外键，引用 `asset_tags.id`，级联删除 |
| `created_at` | TEXT | 非空 |

`session_id + tag_id` 为联合主键，禁止重复标签关联。

### 4.5 会话表调整

`sessions` 表执行以下最终调整：

- 删除字符串字段 `environment`。
- 删除字符串字段 `project`。
- 删除字符串字段 `tags`。
- 保留 `notes`，最大长度 2000 字符。
- 新增可空外键 `environment_id`，引用 `asset_environments.id`。
- 新增可空外键 `project_id`，引用 `asset_projects.id`。

环境和项目删除前必须由服务层完成迁移或清空，因此数据库外键采用受限删除，不允许绕过服务直接产生悬空引用。

## 5. 受控颜色系统

颜色令牌固定为：

`slate`、`red`、`orange`、`amber`、`yellow`、`lime`、`green`、`teal`、`cyan`、`blue`、`violet`、`pink`。

前端在全局 CSS 中为每个令牌定义 Light/Dark Mode 下的背景、文字、边框和状态点颜色。业务数据只保存令牌名称，不保存 Tailwind 类名或 HEX 值。

建议环境默认值：

| 环境 | 默认颜色 |
|---|---|
| 生产 | `red` |
| 预发布 | `orange` |
| 测试 | `amber` |
| 开发 | `blue` |

应用不自动创建默认环境，以上仅作为用户创建同名环境时的推荐颜色。

## 6. 后端服务设计

新增独立的 `AssetCatalogService`，避免继续扩大 `SessionService` 职责。

### 6.1 查询能力

- `ListEnvironments()`：按 `sort_order`、名称排序并返回关联会话数量。
- `ListProjects()`：按 `sort_order`、名称排序并返回关联会话数量。
- `ListTags()`：按名称排序并返回关联会话数量。
- `GetSessionAssetDetail(sessionID)`：返回会话、分组、环境、项目、标签和连接统计。

### 6.2 创建与更新

- `CreateEnvironment(input)`
- `UpdateEnvironment(input)`
- `ReorderEnvironments(ids)`
- `CreateProject(input)`
- `UpdateProject(input)`
- `ReorderProjects(ids)`
- `CreateTag(input)`
- `UpdateTag(input)`

所有名称在服务层统一执行：

1. `TrimSpace`
2. 长度校验
3. 规范化唯一键生成
4. 重复检查
5. 参数化 SQL 写入

### 6.3 删除与迁移

环境和项目删除输入包含：

- 待删除 ID
- 处理模式：`migrate` 或 `clear`
- `migrate` 模式下的目标 ID

事务流程：

1. 锁定并确认源记录存在。
2. 查询关联会话数量。
3. 校验迁移目标存在且不等于源记录。
4. 批量迁移或清空会话引用。
5. 删除源记录。
6. 写入审计日志。
7. 提交事务。

标签删除事务流程：

1. 查询标签及关联数量。
2. 删除 `session_tags` 关联。
3. 删除标签。
4. 写入审计日志。
5. 提交事务。

### 6.4 批量资产操作

新增三个明确接口，避免使用难以区分“未修改”和“清空”的复合输入：

- `BulkSetEnvironment(sessionIDs, environmentID)`：`environmentID` 可空，表示清空。
- `BulkSetProject(sessionIDs, projectID)`：`projectID` 可空，表示清空。
- `BulkUpdateTags(input)`：操作类型为 `add`、`remove` 或 `replace`。

批量接口必须：

- 去重会话 ID 和标签 ID。
- 验证全部引用存在。
- 在单事务中完成。
- 返回受影响会话数量。
- 写入审计日志。

## 7. 会话 API 模型

前端会话模型采用最终结构：

```text
Session
├── environment_id?: number
├── project_id?: number
├── notes: string
└── tags: AssetTag[]
```

列表接口返回环境和项目的轻量展示对象，避免各页面重复查找：

```text
SessionAssetSummary
├── session fields
├── environment?: { id, name, color_token }
├── project?: { id, name, code }
└── tags: [{ id, name, color_token }]
```

环境、项目和标签字典由 `SessionWorkspaceProvider` 统一加载并缓存。创建、更新、删除后只更新对应目录和受影响会话，禁止整页强制刷新。

## 8. 会话创建与编辑

“资产归属”区块调整为：

1. 分组选择
2. 环境可创建单选框
3. 项目可创建单选框
4. 标签可创建多选框
5. 备注

交互规则：

- 环境、项目和标签都允许搜索现有项。
- 输入不存在的名称时提供明确的“创建”操作，不得在失焦时静默创建。
- 快捷创建环境或标签时必须选择颜色。
- 快捷创建项目时可选填项目代号，项目描述在分类管理中维护。
- 所有字段可留空。
- 标签选择结果使用可移除 Badge 展示。

## 9. 资产中心信息架构

资产中心包含四个一级 Tab：

1. 最近连接
2. 分组
3. 所有节点
4. 分类管理

### 9.1 所有节点

页面结构：

```text
[搜索框] [高级筛选] [环境] [项目] [标签]          [创建]
[当前筛选 Chip.........................................]
[批量操作栏，仅选择节点后显示..........................]
--------------------------------------------------------
| 名称 | 端点 | 环境 | 项目 | 标签 | 分组 | 操作 |
--------------------------------------------------------
```

表格规则：

- 环境显示彩色状态点和名称。
- 项目优先显示代号，无代号时显示名称。
- 标签最多显示两个 Badge，剩余显示 `+N`。
- 备注不进入表格。
- 单击行打开详情 Sheet。
- 双击行直接连接。
- 复选框点击不得触发行详情或双击连接。

### 9.2 分类管理

分类管理包含“环境 / 项目 / 标签”子 Tab。

环境列表展示名称、颜色、关联会话数、排序和操作。  
项目列表展示名称、代号、描述摘要、关联会话数、排序和操作。  
标签列表展示名称、颜色、关联会话数和操作。

操作包括创建、编辑、重命名、排序和删除。删除前必须先获取后端影响分析结果。

### 9.3 批量操作

选择会话后显示浮动或表格顶部批量操作栏：

- 设置或清空环境
- 设置或清空项目
- 添加标签
- 移除标签
- 完全替换标签
- 取消选择

所有批量操作在提交前显示目标会话数量。

## 10. 会话详情 Sheet

单击资产表格行，从右侧打开非模态详情 Sheet。

详情分区：

1. 基本信息：名称、主机、端口、用户名、认证方式、分组。
2. 资产信息：环境、项目、标签。
3. 备注：完整展示，空备注显示明确空状态。
4. 使用统计：最近连接时间、连接次数、活动终端数量。
5. 操作：连接、编辑、删除；存在活动终端时提供复制终端入口。

切换表格行时复用同一个 Sheet，不重新播放完整进场动画。Sheet 不遮挡左侧会话树。

## 11. 搜索与筛选

### 11.1 普通搜索

普通搜索覆盖：

- 会话名称
- 主机
- 用户名
- 分组名称
- 环境名称
- 项目名称和项目代号
- 标签名称

普通搜索不包含备注。

### 11.2 高级筛选

高级筛选条件：

- 多选环境
- 多选项目
- 多选标签
- 未设置环境
- 未关联项目
- 无标签
- 最近连接时间范围
- 最小或最大连接次数
- 备注关键词

组合规则：

- 同类别多值使用 OR。
- 不同类别使用 AND。
- 标签条件表示命中任意所选标签。
- 备注关键词只有用户显式添加该条件时才执行匹配。

筛选状态存储在非持久化的应用状态中。离开资产页面后再次返回仍保留；应用重启后重置。

### 11.3 快速连接与侧边栏

- `Ctrl+F` 快速连接搜索增加环境、项目和标签，但不搜索备注。
- 快速连接结果展示环境、项目和最多两个标签。
- 左侧会话树搜索增加环境、项目和标签。
- 会话树 Tooltip 增加环境、项目和标签，继续展示主机、端口和用户。
- Tooltip 和快速连接均不得展示备注。

所有普通搜索入口必须复用同一个规范化匹配工具，禁止各页面分别维护字段列表。

## 12. 字典删除交互

### 12.1 环境与项目

删除对话框展示：

- 名称
- 关联会话数量
- “迁移到其他项”与“清空关联”单选项
- 迁移目标选择器
- 最终影响说明

默认选中“迁移到其他项”。当没有其他可选项时，只允许清空关联。

### 12.2 标签

删除对话框展示标签名称、颜色和关联会话数量，并明确说明确认后将从所有会话移除该标签。

## 13. 同步与备份

同步表清单新增：

- `asset_environments`
- `asset_projects`
- `asset_tags`
- `session_tags`

导出顺序必须先导出字典，再导出会话标签关系。恢复顺序必须先恢复环境、项目、标签，再恢复会话，最后恢复 `session_tags`。

恢复前继续执行严格列校验。因为不支持旧格式，缺少新表或仍包含旧字符串结构的备份直接拒绝导入。

## 14. 安全、性能与可访问性

- 所有 SQL 必须参数化。
- 所有字典名称、代号、描述和备注执行长度限制。
- 备注不得进入快速连接、Tooltip 或默认搜索，减少敏感信息意外暴露。
- 列表筛选和搜索使用预计算的规范化文本，避免每次按键重复拼接全部字段。
- 标签关联更新使用批量 SQL 和事务，不逐条开启事务。
- 所有选择器、Badge 删除按钮、筛选 Chip 和 Sheet 操作支持键盘操作。
- 环境和标签颜色不得作为唯一信息载体，必须同时显示文字。
- 所有颜色组合必须在 Dark/Light Mode 下满足可读性要求。

## 15. EARS 验收清单

验收结果：全部闭环（41/41）。实现已通过前端覆盖率、Go 覆盖率与竞态检测、lint 以及 Wails 全量构建。

### 数据模型

- EARS-ASSET-001：系统应当使用独立环境表存储环境名称、规范化唯一键、颜色和排序。
- EARS-ASSET-002：系统应当使用独立项目表存储项目名称、代号、描述和排序。
- EARS-ASSET-003：系统应当使用独立标签表和会话标签关联表表示多标签关系。
- EARS-ASSET-004：系统不得继续在会话表中保存环境、项目和标签字符串字段。
- EARS-ASSET-005：系统应当允许会话的环境、项目、标签和备注为空。
- EARS-ASSET-006：当名称仅大小写或首尾空格不同时，系统应当将其视为重复字典项。

### 会话编辑

- EARS-ASSET-010：当用户创建或编辑会话时，系统应当允许选择或快捷创建环境、项目和标签。
- EARS-ASSET-011：当用户输入不存在的字典名称时，系统应当要求显式确认创建。
- EARS-ASSET-012：当用户选择多个标签时，系统应当使用可移除 Badge 展示当前选择。
- EARS-ASSET-013：当会话保存失败时，系统应当保留用户已输入的资产信息并展示错误。

### 展示与详情

- EARS-ASSET-020：当资产列表展示会话时，系统应当展示环境、项目、标签和分组。
- EARS-ASSET-021：当标签数量超过两个时，系统应当展示两个标签和剩余数量。
- EARS-ASSET-022：当用户单击会话行时，系统应当打开右侧详情 Sheet。
- EARS-ASSET-023：当用户双击会话行时，系统应当创建 SSH 连接。
- EARS-ASSET-024：当详情 Sheet 展示会话时，系统应当展示完整备注和连接统计。
- EARS-ASSET-025：当会话存在活动终端时，详情 Sheet 应当提供复制终端入口。

### 搜索与筛选

- EARS-ASSET-030：当用户执行普通搜索时，系统应当匹配名称、主机、用户、分组、环境、项目和标签。
- EARS-ASSET-031：普通搜索、快速连接和 Tooltip 不得匹配或展示备注。
- EARS-ASSET-032：当用户显式添加备注高级条件时，系统应当匹配备注关键词。
- EARS-ASSET-033：当同类别选择多个筛选项时，系统应当使用 OR 规则。
- EARS-ASSET-034：当不同类别同时存在筛选项时，系统应当使用 AND 规则。
- EARS-ASSET-035：系统应当支持筛选未设置环境、未关联项目和无标签会话。
- EARS-ASSET-036：当应用未重启时，系统应当保留资产页面筛选状态。
- EARS-ASSET-037：当应用重新启动时，系统应当清空上一次资产筛选状态。

### 字典管理

- EARS-ASSET-040：系统应当在分类管理中提供环境、项目和标签的创建、编辑与删除能力。
- EARS-ASSET-041：当环境或项目仍被会话使用时，系统不得直接删除该记录。
- EARS-ASSET-042：当删除被使用的环境或项目时，系统应当要求迁移或清空关联。
- EARS-ASSET-043：当删除标签时，系统应当展示关联数量并要求确认。
- EARS-ASSET-044：当重命名字典项时，所有关联会话应当立即展示新名称。
- EARS-ASSET-045：所有字典迁移和删除操作应当在单个数据库事务中执行。
- EARS-ASSET-046：所有字典变更、迁移和删除操作应当写入审计日志。

### 批量操作

- EARS-ASSET-050：当用户选择一个或多个会话时，系统应当显示批量资产操作栏。
- EARS-ASSET-051：系统应当支持批量设置或清空环境和项目。
- EARS-ASSET-052：系统应当支持批量添加、移除和完全替换标签。
- EARS-ASSET-053：批量操作应当在提交前展示目标会话数量。
- EARS-ASSET-054：批量操作应当在单事务中完成，失败时不得产生部分更新。

### 同步与质量

- EARS-ASSET-060：同步导出应当包含环境、项目、标签和会话标签关系。

- EARS-ASSET-062：旧格式备份应当被明确拒绝，不得执行兼容转换。
- EARS-ASSET-063：所有新增后端方法应当覆盖成功、重复、无效输入、迁移、清空和回滚测试。
- EARS-ASSET-064：所有新增前端交互应当覆盖键盘操作、筛选组合、空状态和错误状态测试。
- EARS-ASSET-065：最终实现应当通过前端覆盖率、Go 测试、竞态检测、lint 和 Wails 全量构建。
